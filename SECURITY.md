# Security Policy

## Supported Versions

Only the latest release receives security fixes. Pre-v1.0 has no stability guarantees,
but vulnerability reports are still welcome.

## Reporting a Vulnerability

Use GitHub's private vulnerability reporting:

https://github.com/wood-jp/xerrors/security/advisories/new

Please include:
- xerrors version
- Go version
- Description of the vulnerability
- Steps to reproduce
- Impact assessment

Do not open a public issue for security vulnerabilities.

## Response

We aim to acknowledge reports within a few business days. Fix timelines are
best-effort — this is a small open-source project.

## Disclosure

Once a fix is released, a GitHub Security Advisory will be published.
